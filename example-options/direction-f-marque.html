Analytics

03

Model Context Protocol · the pillar the name argues for

A letter of marque for every agent.

A letter of marque was a document granting one party authority to act for another — within stated limits, for a stated time, on the public record. That is precisely what a delegated token is.

Marque treats an AI agent as a principal in its own right. It never borrows a human session, never inherits a human's full rights, and never acts without a paper trail.

01Scoped

A grant names the MCP server, the exact tools, and the ceilings — calls, rows, spend. Anything unnamed is denied. There is no wildcard.

02Time-bound

Grants carry a clock, and the default is minutes. An agent that finishes its work holds nothing afterwards. Nothing sits in a config file forever.

03Auditable

The delegation chain is recorded on every call: which person, which agent, which tool, which rows. An answer to "who read this" takes one query, not one week.

04Revocable

One call kills a grant, a whole agent, or every agent acting for one person. Revocation reaches the edge in under a second, because verification checks the live grant.

Authorise the tool call, not the session. That is the whole difference.

Two grants, in full

Verify a person. Commission an agent.

Verify an OIDC access token — any Node framework

// Marque publishes discovery and JWKS where clients look.
import { createVerifier } from "@marque/verify";

const marque = createVerifier({
  issuer:   "https://auth.acme.internal",
  audience: "orders-api",
  clock:    { skewSeconds: 30 },
});

export async function guard(req: Request) {
  const claim = await marque.verify(req.headers.get("authorization"));

  if (!claim.ok)                          return deny(401, claim.reason);
  if (!claim.scopes.has("orders:write")) return deny(403, "scope");

  // "user:8f21", or "agent:planner-3" acting for one.
  return claim.principal;
}

Ask the stream

Questions in plain words. Answers from your own log.

who read invoices in the last hour?

Four principals. Three people, one agent. The agent read 212 rows under a grant that expired at 14:20 and was not renewed.

anything odd about the billing service today?

It asked for refunds:write at 03:11, the first time in ninety days. Policy denied it, the event was flagged, and a ticket is open with the request id attached.

which grants do agents still hold?

Nine live grants across two tenants. Two expire within the hour. One has never been used and can be dropped without breaking anything.

14:02:11 grant.issued agent:planner-3 · 14:02:12 tool.call invoices.read rows=48 · 14:02:19 policy.deny scope=refunds:write · 14:02:19 risk.score 0.91 flagged · 14:02:20 grant.revoked grn_7QW3d1x

The model runs inside your deployment, against your event store. No log line is sent anywhere for scoring, ever.

Self-hosting

Your hardware. Your data. Your audit trail.

Marque is a single binary and a PostgreSQL database. If you can run those, you can run your own identity provider, and you can keep running it when your network is cut off from the world.

  • LaptopOne compose file brings up the server, the database and a seeded tenant. It is ready before your coffee is.
  • ClusterA Helm chart with sane defaults, horizontal scaling, and rolling upgrades that do not drop a live session.
  • Air-gappedNo outbound calls, no licence check, no telemetry. The install works with the cable unplugged.
  • KeysSigning keys live in your KMS or on your disk. Rotation is scheduled and old keys stay published until the last token expires.
  • BackupsA backup is a pg_dump. A restore is a pg_restore. There is no proprietary state hiding elsewhere.
  • MigrationImporters read realms, users and clients from Keycloak or Okta, so moving is a weekend rather than a quarter.
$ marque up --tenant acme --oidc --saml --mcp $ marque import --from keycloak --realm acme --dry-run $ marque grants revoke --agent planner-3 --all
