▌ pillars ──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────── three surfaces, one binary

01authorization

The part other people integrate against

Standard protocols, not a proprietary SDK. If a vendor speaks OIDC or SAML, it speaks Bastion.

├── oidc auth code + PKCE, device, CIBA ├── oauth 2.1 DPoP-bound tokens, no implicit ├── saml 2.0 SP- and IdP-initiated, encrypted ├── scim 2.0 provisioning, JIT deprovision ├── policy ReBAC tuples, /v1/check p50 1.8ms ├── tenancy row-level, per-tenant signing keys └── machines client_credentials, API keys
api keys hashed sha-256, indexed by prefix · RFC 9700, RFC 9449
02signal

The auth stream is the dataset

Every decision is an event with a subject, a tuple path and a verdict. The model reads that stream, never your application database.

├── baseline per principal, 14-day rolling ├── anomaly geo velocity, replay, scope drift ├── risk 0.00-1.00, carried in the token ├── ask plain-language queries over audit ├── respond revoke, quarantine, page on-call ├── explain every score cites the events └── local inference in your cluster
bundled 3B model, or point it at an endpoint you control
03agents / mcp

An agent is a principal, not a shared key

Model Context Protocol servers get scoped, delegated, expiring sessions. Who asked, on whose behalf, and what was allowed are all recorded.

├── principal agt_* identity, own audit trail ├── delegate on-behalf-of a human, consent kept ├── scope per-tool allow list, deny default ├── approve step-up before flagged tool calls ├── budget call ceilings per grant, per hour ├── expire minutes, not months └── revoke one command kills live sessions
mcp over stdio and streamable http · tool calls signed and replayable

▌ integrate ──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────── 2 files, both real

~/api/verify-token.tsresource server
import { createVerifier } from "@bastion/verify"; // JWKS is cached and rotated on kid miss. // No network call in the request path. const verify = createVerifier({ issuer: "https://id.acme.example/o/acme", audience: "api.internal", algorithms: ["ES256"], clockToleranceSec: 30, }); export async function requireScope(req, scope) { const bearer = req.header("authorization")?.slice(7); const claims = await verify(bearer, { dpop: req.header("dpop"), // proof of possession htu: req.url, htm: req.method, }); if (claims.tenant !== req.tenantId) throw new Forbidden("tenant_mismatch"); if (!claims.scope.split(" ").includes(scope)) throw new Forbidden("insufficient_scope"); return claims; // sub, tenant, scope, risk, jti, exp }
verification is local · a revoked session fails the next check, not the current one
~/agents/grant-session.tsmcp host
import { Bastion } from "@bastion/sdk"; const bastion = new Bastion({ url: "https://id.acme.example", }); // The agent is a principal. The grant is scoped, // delegated, budgeted and expiring. const session = await bastion.mcp.grant({ agent: "agt_5f21", // claude-ops onBehalfOf: "u_8817", // consent stored tools: ["tickets.read", "deploys.status"], requireApproval: ["deploys.*"], budget: { calls: 200, perHour: 60 }, ttl: "30m", }); session.on("call", (call) => { audit.write(call); // tool, args hash, decision }); // Kill it early, from anywhere: await session.revoke("task_complete");
grants are tokens, not config · nothing here is stored on the agent host
